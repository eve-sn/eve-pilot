Chargée de suivi de projet {% if 'chargee_suivi' in primary_sections %}VOUS{% endif %}

Pilote l'avancement du projet, crée les demandes de dépense, soumet les rapports terrain. Voit uniquement son ou ses projet(s).

Au quotidien

  1. Vous connecter à {{ site_base_url }}/connexion/
  2. Créer une demande de dépense : Finance → Demandes → "Nouvelle demande"
    • Sélectionner votre projet
    • Choisir la ligne budgétaire
    • Saisir titre, motif, montant
    • Joindre la proforma ou le contrat (section "Pièces justificatives")
    • Cliquer "Soumettre pour validation"
  3. Soumettre un rapport terrain : Activités → ouvrir l'activité → "+ Rapport terrain"
    • Date, lieu, participants (H/F/enfants), narratif, résultats, difficultés, recommandations
    • Ajouter les bénéficiaires nominatifs (nom, genre, commune)
    • Téléverser les preuves (photos, liste de présence signée)
  4. Surveiller le bandeau "Notifications" sur Finance : "brouillon à soumettre", "rejeté à retravailler"

Ce que vous ne pouvez pas faire

  • Voir les autres projets que les vôtres
  • Voir le Budget Général, le plan comptable, le bilan
  • Valider une demande de dépense
  • Saisir un paiement (sauf si vous êtes la demandeuse de cette dépense précise)

Comptable {% if 'comptable' in primary_sections %}VOUS{% endif %}

Prépare, exécute et enregistre les paiements. Tient la comptabilité au jour le jour pour les projets de son périmètre.

Au quotidien

  1. Vous connecter à {{ site_base_url }}/connexion/
  2. Bandeau "Notifications" sur le Tableau Finance : il indique combien de demandes attendent l'exécution
  3. Saisir un paiement :
    • Ouvrir la demande approuvée
    • Vérifier qu'une facture définitive est jointe — sinon le bouton est bloqué
    • Cliquer "Saisir la dépense"
    • Renseigner : mode (banque/caisse), compte, date d'opération, référence relevé, montant réel, compte de charge SYCEBNL, commentaire
    • Submit → le BankMovement et l'écriture comptable sont créés automatiquement
  4. Consulter la trésorerie : Finance → Trésorerie (plan mensuel des projets accessibles)
  5. Ouvrir les fiches de chaque compte bancaire rattaché à ses projets

Ce que vous voyez

  • Les projets de votre périmètre (5 projets pour SAKHO : Nous-Cims SL, ECP, PIK-MBAO, GT-WALLU, ONAS-PDBH)
  • Toutes leurs demandes de dépense, mouvements bancaires, lignes budgétaires

Ce que vous ne pouvez pas faire

  • Valider une demande (vous n'avez pas le rôle validateur)
  • Voir le plan comptable, la balance, le bilan (réservé RAF/DP)
  • Voir le Budget Général

Assistante RAF — Budget Général et petite caisse {% if 'araf' in primary_sections %}VOUS{% endif %}

Gère la comptabilité du Budget Général et la petite caisse de fonctionnement (femmes de charge, stagiaires, restauration personnel, réception et hospitalités des hôtes et invités).

Au quotidien

  1. Vous connecter à {{ site_base_url }}/connexion/
  2. Créer une demande de dépense BG : Finance → Demandes → "Nouvelle demande"
    • Sélectionner "Budget Général (aucun projet)"
    • Choisir la ligne BG correspondante (charges fixes, salaires personnel d'appui, etc.)
    • Joindre la pièce justificative
    • Soumettre pour validation aux 3 valideurs (RAF, DP, SE)
  3. Saisir un paiement BG approuvé :
    • Ouvrir la demande approuvée
    • Vérifier la facture jointe
    • Cliquer "Saisir la dépense"
    • Choisir le mode : Caisse pour les petites dépenses, Banque pour les virements (salaires personnel d'appui, factures fournisseurs)
    • Renseigner date, référence, montant réel, compte de charge SYCEBNL
  4. Surveiller le bandeau "Notifications" sur le tableau de bord Finance

Ce que vous voyez

  • Budget Général : toutes les lignes BG, demandes BG, comptes bancaires BG (BG SUNU notamment), petite caisse
  • Tableau de bord finance avec section BG mise en avant

Ce que vous ne pouvez pas faire

  • Voir les projets bailleurs et leurs comptes (ce n'est pas votre périmètre)
  • Valider une demande de dépense (vous n'avez pas le rôle validateur)
  • Voir le plan comptable, la balance, le bilan (réservé RAF/DP)

RAF — Responsable Administratif et Financier {% if 'raf' in primary_sections %}VOUS{% endif %}

Première signature du workflow dépense. Vue globale finance, accès comptable technique complet.

Au quotidien

  1. Signer les demandes : bandeau dashboard "X demande(s) à signer (RAF)" → ouvrir, vérifier projet/ligne/montant/justificatifs, signer Approuver ou Rejeter (avec commentaire si rejet)
  2. Vérifier la balance comptable mensuellement : Finance → Balance
  3. Suivre le bilan : Finance → Bilan
  4. Compte de résultat : suivi charges/produits par classe SYCEBNL
  5. Vous pouvez exécuter n'importe quelle demande approuvée (back-up si le comptable est indisponible)

Ce que vous voyez

  • Tout : 10 projets, Budget Général, comptes bancaires, plan comptable, balance, compte de résultat, bilan, grand livre par compte

DP — Directrice des Programmes {% if 'dp' in primary_sections %}VOUS{% endif %}

Deuxième signature du workflow. Valide la pertinence métier des dépenses et supervise les activités terrain.

Au quotidien

  1. Signer les demandes en DP : bandeau dashboard ou Finance → Demandes → ?to_sign=1
  2. Vérifier la cohérence avec l'activité prévue (description du motif vs cadre logique)
  3. Suivre les activités terrain : Activités → liste filtrable par projet, statut, type
  4. Consulter les rapports terrain validés pour préparer les rapports bailleurs

Ce que vous voyez

  • Vue globale comme le RAF, sauf que vous ne saisissez pas les paiements (ce n'est pas votre rôle métier)

SE — Secrétaire Exécutif {% if 'se' in primary_sections %}VOUS{% endif %}

Troisième signature du workflow dépense. Validation finale des rapports d'activité terrain. Pilotage organisationnel (sans plongée dans les états comptables).

Au quotidien

  1. Signer la ligne SE des demandes soumises (bandeau "à signer (SE)")
  2. Valider les rapports terrain : Activités → ouvrir un rapport soumis → décider VALIDE ou REJETÉ (commentaire obligatoire pour rejeter)
  3. Surveiller le tableau de bord finance pour la vue d'ensemble organisationnelle

Ce que vous voyez

  • Vue globale projets, demandes, activités, trésorerie

Ce que vous ne voyez pas

  • Plan comptable, balance, compte de résultat, bilan, grand livre — ces écrans techniques sont réservés RAF/DP

Administrateur technique {% if 'admin' in primary_sections %}VOUS{% endif %}

Maintenance, création de comptes, débogage, sauvegardes.

Actions exclusives

  1. Console Django Admin : /admin/ pour créer des Users, attribuer des rôles UserRole, gérer les référentiels
  2. Surveiller le serveur Waitress (console PowerShell active)
  3. Lancer une sauvegarde manuelle : .\scripts\backup_uat.ps1
  4. Consulter les logs d'erreur

Règle d'or

  • Ne pas valider de demandes avec ce compte. Les rapports d'audit doivent tracer les vraies personnes métier, pas un super-user technique. Utilisez votre compte applicatif personnel pour valider.

Workflow complet d'une demande de dépense

Cycle DRAFT → SUBMITTED → APPROVED → EXECUTED.

  1. Création : la chargée de suivi crée la demande, joint la proforma. Statut = Brouillon.
  2. Soumission : elle clique "Soumettre pour validation". 3 lignes de validation (RAF, DP, SE) sont créées. Statut = Soumise. Les 3 valideurs reçoivent une notification.
  3. Validations : chaque valideur signe sa ligne. Ordre indifférent.
  4. Approbation : à la 3ème signature Approuvée, statut → Approuvée. La demandeuse et le comptable sont notifiés.
  5. Rejet : un seul Rejet (avec commentaire) → statut Rejetée. La demandeuse peut retravailler.
  6. Pièces de paiement : la facture définitive est jointe avant exécution.
  7. Exécution : le comptable clique "Saisir la dépense", renseigne le BankMovement, valide. Statut → Exécutée. L'écriture SYCEBNL est générée automatiquement (débit charge, crédit banque).

Règles d'usage — rappel rapide

  • Ne JAMAIS partager votre mot de passe
  • Vous déconnecter en quittant le poste : /deconnexion/
  • Toute donnée bénéficiaire est confidentielle — ne pas exporter, ne pas photographier l'écran
  • Signaler immédiatement à l'administrateur tout comportement étrange de l'application
  • En cas de doute sur une demande, demander à voir la pièce physique avant de signer
